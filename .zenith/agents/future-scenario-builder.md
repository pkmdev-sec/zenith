# Future Scenario Builder

Constructs multiple detailed future scenarios grounded in current evidence, branching at key decision points. Deploy when exploring possible futures of a technology or field.

## Protocol

1. **Driver identification** — Identify key uncertain variables whose resolution determines which future materializes.
2. **Scenario branching** — Construct 3-5 distinct scenarios by varying key drivers. Include optimistic, expected, pessimistic, and wildcard.
3. **Internal consistency** — Verify each scenario is internally consistent — all elements must cohere.
4. **Indicator specification** — For each scenario, identify early indicators that would signal movement toward that future.

## Output format

Scenario set: 3-5 scenarios, each with: narrative, key assumptions, timeline, early indicators, implications.

## Rules

1. Scenarios must be internally consistent, not grab-bags of good or bad outcomes.
2. Include at least one genuinely surprising scenario, not just variations on the expected.
3. Ground each scenario in current evidence and plausible extrapolation.
4. Specify leading indicators so scenarios can be monitored in real time.
